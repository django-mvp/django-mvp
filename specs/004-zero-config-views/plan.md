# Implementation Plan: Zero-Config Ready-to-Use Views

**Branch**: `004-zero-config-views` | **Date**: 2026-05-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-zero-config-views/spec.md`

## Summary

Ship two zero-config ready-to-use views: `MVPTemplateView` for plain layout-aware pages, and `MVPHomeView` for a dual-purpose landing/dashboard from a single URL. Both already exist in `mvp/views/base.py` — this feature closes the remaining gaps: attribute naming consistency, the `ImproperlyConfigured` guard for a missing dashboard template, creation of the bundled `mvp/landing.html` and `mvp/dashboard.html` base templates, and complete test coverage.

## Technical Context

**Language/Version**: Python 3.11+
**Primary Dependencies**: Django 4.2+, django-cotton (Cotton component syntax), django-cotton-bs5 (prebuilt Bootstrap 5 components)
**Storage**: N/A — both views are stateless and query-free
**Testing**: pytest, pytest-django (unit/integration), pytest-playwright (E2E)
**Target Platform**: Any Django 4.2+ WSGI/ASGI project using the `mvp` package
**Project Type**: Reusable Django app (single source tree)
**Performance Goals**: Zero DB queries per request for both views in their default configuration
**Constraints**: No model, form, or queryset configuration required; GET-only; anonymous access always permitted
**Scale/Scope**: Two view classes, two bundled templates, ~50 lines of new production code (guards + templates + alias)
