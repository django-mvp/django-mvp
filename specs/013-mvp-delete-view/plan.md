# Implementation Plan: MVP Delete View

**Branch**: `013-mvp-delete-view` | **Date**: 2026-05-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/013-mvp-delete-view/spec.md`

## Summary

`MVPDeleteView` is a configurable AdminLTE-integrated delete view that handles four deletion scenarios — simple confirmation, related-objects summary, protected-record detection, and type-to-confirm — from a single class. The design maximises reuse of Django 5.x's `DeleteView` form machinery (`FormMixin` + `DeletionMixin`): `get_form_class()` selects `DeleteConfirmForm` when type-to-confirm is active; `form_valid()` / `form_invalid()` manage the success/failure paths; `post()` performs the protection check before form processing. Custom code in `MVPDeleteView` is kept to a minimum by leveraging the `MVPModelFormBase` inheritance chain for page title, success message, CRUD directory, breadcrumbs, and redirect resolution.

## Technical Context

**Language/Version**: Python 3.12, Django 5.x
**Primary Dependencies**: django-mvp (internal), django-cotton-bs5 (template components), Bootstrap 5 (CSS)
**Storage**: Django ORM (SQLite in development, PostgreSQL in production) — no new migrations
**Testing**: pytest, pytest-django, pytest-playwright, factory-boy
**Target Platform**: Django web application (server-rendered)
**Project Type**: Reusable Django library
**Performance Goals**: At most two `Collector` traversals per request (GET and POST→invalid). No caching needed — delete pages are low-traffic and the Collector traversal is bounded by the model's FK graph.
**Constraints**: Must not crash with an unhandled exception on `PROTECT` FK conflicts. Must not use `object.get_absolute_url()` as a post-delete redirect fallback.
**Scale/Scope**: Single class (`MVPDeleteView`) + one form (`DeleteConfirmForm`) + one template (`delete_view.html`) update.

## Phase 0: Research

All unknowns resolved. See [research.md](research.md) for full decision rationale.

**Summary of key decisions**:

1. **Django 5.x form machinery** — Use `get_form_class()` / `get_form_kwargs()` / `form_valid()` / `form_invalid()` pipeline instead of monolithic `post()` override. Reduces `MVPDeleteView` to only the delta from `MVPModelFormBase`.
2. **`DeleteConfirmForm`** — Move confirmation-value match into `clean_confirmation()` with `confirmation_value` injected via `__init__`. `form_valid()` is then unconditionally safe to delete.
3. **Django `Collector`** — Used without caching; at most two traversals per request, acceptable for a delete page.
4. **Display cap** — `(label, display_list, overflow_count)` triples passed to template; view-layer truncation; template renders "… and N more".
5. **`SuccessMessageMixin`** — Not invoked through super-chain; `messages.success()` called directly in `form_valid()`.
6. **Post-delete redirect** — Priority: `?next=` → `success_url` → list URL. Object URL never used.
7. **Breadcrumbs** — List → Detail → Delete, both links gated by permission flags.

### `MVPDeleteView` — implementation sketch

```python
class MVPDeleteView(MVPModelFormBase, generic.DeleteView):
    base_template_name = "delete_view.html"
    page_title = _("Delete %(verbose_name)s")
    page_icon = "delete"
    page_class = "mvp-delete-page"
    success_message = _("%(verbose_name)s successfully deleted.")

    show_related_objects: bool = False
    require_confirmation: bool = False
    confirmation_label: str = _("Type the name to confirm")
    related_objects_max_per_group: int = 25

    def get_confirmation_value(self) -> str:
        return str(self.object)

    def get_back_url(self) -> str:
        candidate = self.request.GET.get("back")
        if candidate and url_has_allowed_host_and_scheme(...):
            return candidate
        return self.resolve_crud_url("list") or ""

    def get_breadcrumbs(self):
        return [
            {"text": self.get_list_title(), "href": self.resolve_crud_url("list")},
            {"text": str(self.object), "href": self.resolve_crud_url("detail")},
            {"text": self.get_page_title()},
        ]

    def get_form_class(self):
        if self.require_confirmation:
            return DeleteConfirmForm
        return super().get_form_class()

    def get_form_kwargs(self):
        kwargs = super().get_form_kwargs()
        if self.require_confirmation:
            kwargs["confirmation_value"] = self.get_confirmation_value()
        return kwargs

    def _collect_deletion_data(self):
        # Collector.collect() → (related_map, protected_objects)
        ...

    def post(self, request, *args, **kwargs):
        self.object = self.get_object()
        _, protected = self._collect_deletion_data()
        if protected:
            return self.render_to_response(self.get_context_data())
        form = self.get_form()
        if form.is_valid():
            return self.form_valid(form)
        return self.form_invalid(form)

    def form_valid(self, form):
        success_url = self.get_success_url()
        self.object.delete()
        messages.success(self.request, self.get_success_message({}))
        return HttpResponseRedirect(success_url)

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        related_map, protected = self._collect_deletion_data()
        context["is_protected"] = bool(protected)
        context["protected_objects"] = protected
        context["require_confirmation"] = self.require_confirmation
        context["confirmation_value"] = self.get_confirmation_value() if self.require_confirmation else ""
        context["confirmation_label"] = self.confirmation_label
        context["back_url"] = self.get_back_url()
        if self.show_related_objects and not protected:
            cap = self.related_objects_max_per_group
            context["related_objects"] = [
                (model._meta.verbose_name_plural.title(), list(instances)[:cap], max(0, len(instances) - cap))
                for model, instances in related_map.items()
            ]
        else:
            context["related_objects"] = []
        return context

    def get_success_url(self):
        # Inherit steps 1-3 from MVPModelFormBase (next URL → success_url shorthand → success_url literal).
        # Override only to replace step 4 (object.get_absolute_url — 404 after deletion)
        # with the list URL.
        try:
            return super().get_success_url()
        except ImproperlyConfigured:
            url = self.resolve_crud_url("list")
            if url:
                return url
            raise
```

### `DeleteConfirmForm` — updated

```python
class DeleteConfirmForm(forms.Form):
    confirmation = forms.CharField(
        label=_("Confirmation"),
        widget=forms.TextInput(attrs={"autocomplete": "off"}),
    )

    def __init__(self, *args, confirmation_value=None, **kwargs):
        super().__init__(*args, **kwargs)
        self._confirmation_value = confirmation_value

    def clean_confirmation(self):
        value = self.cleaned_data["confirmation"].strip()
        if self._confirmation_value is not None and value != self._confirmation_value:
            raise forms.ValidationError(
                _("The value you entered does not match. Please try again.")
            )
        return value
```
