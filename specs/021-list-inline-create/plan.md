# Implementation Plan: List View Inline Create

**Branch**: `021-list-inline-create` | **Date**: 2026-06-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/021-list-inline-create/spec.md`

## Summary

Add `create_form_class` and `create_modal_title` attributes to `MVPListViewMixin` so that list
views can surface an inline create form inside a Bootstrap 5 modal without requiring a separate
page navigation. When the attribute is set and the user has create permission, the mixin injects
an unbound form instance (`create_form`) and a resolved title (`create_modal_title`) into the
template context. The list template modal is generalised to use these context values; the form
action appends `?next={{ request.path }}` so that `MVPCreateView`'s existing `NextURLMixin`
redirects the user back to the list after a successful create.

### Phase 1 — Mixin Extension (`MVPListViewMixin`)

**Files**: `mvp/views/list.py`
**User Stories**: US1 (partial), US2, US3

**Changes**:

1. Add class attributes to `MVPListViewMixin`:

   ```python
   create_form_class = None       # Django form class or None
   create_modal_title = None      # str override or None (auto-derives from verbose_name)
   ```

2. Add `get_create_form()` hook:

   ```python
   def get_create_form(self):
       """Return an unbound create form instance, or None if not configured."""
       if self.create_form_class is None:
           return None
       return self.create_form_class()
   ```

3. Extend `get_context_data()` to inject `create_form` and `create_modal_title` when
   `create_form_class` is set and permission is granted:

   ```python
   perm = self.has_create_permission
   allowed = perm(self.request.user) if callable(perm) else bool(perm)
   if allowed and self.create_form_class:
       context["create_form"] = self.get_create_form()
       title = self.create_modal_title or f"Add {self.model._meta.verbose_name.title()}"
       context["create_modal_title"] = title
   ```

4. Update the `MVPListViewMixin` docstring to document:
   - `create_form_class`, `create_modal_title` in Config section
   - `get_create_form()` in Override hooks section
   - `create_form`, `create_modal_title` in Context section

**Validation**:

- `poetry run python manage.py check`
- `poetry run pytest tests/test_views/test_list_view.py -x -q`
- `poetry run ruff check mvp/views/list.py`

### Phase 2 — Template Generalisation (`list_view.html`)

**Files**: `mvp/templates/list_view.html`
**User Stories**: US1 (complete)

**Changes**:

1. Wrap the existing `<c-modal id="createModal">` block in `{% if create_form %}...{% endif %}` to prevent it rendering when the form is not in context.

2. Replace the hardcoded title:

   ```html
   title="{% trans "Create product" %}"
   ```

   with:

   ```html
   title="{{ create_modal_title }}"
   ```

3. Append `?next={{ request.path }}` to the form action:

   ```html
   action="{{ directory.create_url }}?next={{ request.path }}"
   ```

4. Run djlint to verify no template linting violations:

   ```
   poetry run djlint mvp/templates/list_view.html --check
   ```
