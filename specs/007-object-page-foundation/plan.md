## Summary

`PageObjectMixin` and `MVPDetailView` are the shared composition layer for all object-level views in django-mvp. `PageObjectMixin` merges three prior building blocks — `ModelInfoMixin` (model resolution), `CRUDDirectoryMixin` (permission-gated sibling URLs), and `PageMixin` (page header/breadcrumbs) — into a single base. `MVPDetailView` is the simplest concrete result: a read-only page whose title comes from `str(self.object)`, whose breadcrumb trail leads back to the model's list view, and whose page container carries both a model-scoped class and an action-specific class.

The full implementation already exists in `mvp/views/detail.py` and satisfies all 9 functional requirements without modification. Research (see `research.md`) confirms the only gaps are:

1. **Zero unit test coverage** for `PageObjectMixin` and `MVPDetailView`.
2. **No E2E test** for the `ProductDetailView` demo verifying the heading, breadcrumbs, and CSS classes in a real browser.
3. **No quickstart doc** for developers.
4. **`skills/django-mvp/SKILL.md`** does not yet document `MVPDetailView` or `PageObjectMixin` as part of the public API.
