# Implementation Plan: Form View Base Classes

**Branch**: `009-form-view-base` | **Date**: 2026-05-04 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/009-form-view-base/spec.md`

## Summary

`MVPFormBase` and `MVPModelFormBase` are the shared foundation for all form views in the
package. Research confirms that 8 of 10 functional requirements are already satisfied by
the existing implementation. Two targeted code changes are required:

1. **FR-007** ‚ÄĒ `MVPModelFormBase.get_success_message()`: wrap the interpolation dict in
   `collections.defaultdict(str, ...)` so missing field-value placeholders (e.g. `%(name)s`
   on a delete view) substitute `""` instead of raising `KeyError`.
2. **FR-008** ‚ÄĒ `MVPModelFormBase.get_success_url()`: redesign the fallback chain so that
   (a) `success_url` is first tried as a CRUD shorthand key via `resolve_crud_url()` ‚ÄĒ
   enabling `success_url = "list"` ‚ÄĒ and (b) when all earlier steps fail, the view falls
   back to `self.object.get_absolute_url()` instead of `resolve_crud_url("list")`, raising
   `ImproperlyConfigured` only when the object lacks `get_absolute_url`.

Seven existing tests cover FR-007 and the class-level attribute contracts. Three additional
tests are required for the redesigned FR-008 behaviour (T022‚ÄďT024) plus one implementation
task (T025).

## Phase 1: Design

### Change 1 ‚ÄĒ `MVPModelFormBase.get_success_message()` (FR-007)

**File**: `mvp/views/edit.py`

**Before**:

```python
def get_success_message(self, cleaned_data):
    return self.success_message % {
        **cleaned_data,
        "verbose_name": self.model_meta.verbose_name,
    }
```

**After**:

```python
from collections import defaultdict

def get_success_message(self, cleaned_data):
    data = defaultdict(str, cleaned_data)
    data["verbose_name"] = self.model_meta.verbose_name
    return self.success_message % data
```

`defaultdict(str, cleaned_data)` pre-populates the dict with all form field values (if
any). Any placeholder key absent from `cleaned_data` (e.g. `%(name)s` on a delete view)
returns `""` via the default factory. `%(verbose_name)s` is always injected explicitly
after construction, ensuring it takes precedence if a form field happened to be named
`verbose_name`.

**Backward compatibility**: Existing success messages using only `%(verbose_name)s` are
unaffected ‚ÄĒ `defaultdict(str)` behaves identically to a plain `dict` for keys that are
present.

### Change 2 ‚ÄĒ `MVPModelFormBase.get_success_url()` (FR-008)

**File**: `mvp/views/edit.py`

**Before** (current implementation ‚ÄĒ shipped in first iteration of this feature):

```python
def get_success_url(self):
    try:
        return super().get_success_url()
    except ImproperlyConfigured:
        url = self.resolve_crud_url("list")
        if url is None:
            raise ImproperlyConfigured(
                f"'{self.__class__.__name__}' could not resolve a list URL. "
                f"Configure 'crud_views' or set 'success_url'."
            )
        return url
```

**After**:

```python
def get_success_url(self):
    # Step 1: validated next URL / CRUD shorthand (inherited from MVPFormBase)
    if next_url := self.get_next_url():
        return next_url

    # Step 2: success_url ‚ÄĒ tried first as a CRUD shorthand, then as a literal path
    raw = getattr(self, "success_url", None)
    if raw:
        resolved = self.resolve_crud_url(str(raw))
        if resolved:
            return resolved
        return str(raw)  # treat as a direct URL path

    # Step 3: object.get_absolute_url() final fallback
    obj = getattr(self, "object", None)
    if obj is not None and callable(getattr(obj, "get_absolute_url", None)):
        return obj.get_absolute_url()

    raise ImproperlyConfigured(
        f"'{self.__class__.__name__}' could not determine a redirect URL. "
        f"Set 'success_url' (e.g. \"list\"), or ensure the model defines "
        f"'get_absolute_url()'."
    )
```

**Priority chain documented**:

1. Validated `next` URL / CRUD shorthand from request (inherited from `MVPFormBase`)
2. `success_url` ‚ÄĒ first attempted as `resolve_crud_url(success_url)`; if that returns
   `None` or `success_url` is not a known shorthand, treated as a direct URL path
3. `self.object.get_absolute_url()` ‚ÄĒ final zero-config fallback for model views
4. `ImproperlyConfigured` ‚ÄĒ raised when object is absent or lacks `get_absolute_url`

**Note for delete views**: Step 3 will compute the object's URL after deletion, producing
a 404 on the next request. Delete views should set `success_url = "list"` explicitly.

**Backward compatibility**: Views that already set `success_url` as a literal path are
unaffected ‚ÄĒ the literal-path branch returns the same value as before. Views relying on
the old `resolve_crud_url("list")` automatic fallback now need either `success_url = "list"`
or a model with `get_absolute_url`.
