**Input**: Feature specification from `/specs/005-model-resolution-mixin/spec.md`

## Summary

`ModelInfoMixin` provides a single, canonical model-resolution mechanism that works regardless of which Django configuration style a downstream view uses (direct model attribute, queryset, ModelForm class, or pre-fetched object instance). The current implementation in `mvp/views/detail.py` already satisfies the four-strategy resolution chain, the override point (`get_model_class()`), the `model_info` context key, and the diagnostic error message. The sole code change this feature delivers is **moving `ModelInfoMixin` from `mvp.views.detail` to `mvp.views.base`** alongside `BaseTemplateNameMixin` and `PageMixin`. The primary work is **comprehensive test coverage** — no tests exist today for any of the mixin's resolution strategies, edge cases, or error behavior.
