{% extends "page_view.html" %} {% block page.actions %} {% endblock page.actions %} {% block page.content %} {% comment %} Two override entrypoints. Create one or both of these files in your app's static/ folder:
  • bootstrap_variables.scss — Bootstrap design tokens ($primary, $gray-800, …). Loaded before all vendor defaults.
  • adminlte_variables.scss — AdminLTE-specific variables, including those that reference Bootstrap tokens (e.g. $lte-sidebar-color: $primary). Injected after Bootstrap resolves but before AdminLTE's own defaults.
{% endcomment %} {# ------------------------------------------------------------------ #} {# INSTALLED_APPS #} {# ------------------------------------------------------------------ #}

Django's static-files finder searches INSTALLED_APPS in order and stops at the first match. Place your app before mvp so your override files shadow the defaults: 

# settings.py
INSTALLED_APPS = [
    ...
    "your_app",   # ← before mvp — your static/ files take priority
    "mvp",
    ...
]
 {# ------------------------------------------------------------------ #} {# ENTRYPOINT 1 #} {# ------------------------------------------------------------------ #}

Create this file in your app's static/ folder: 

your_app/
└── static/
    └── bootstrap_variables.scss

Use it for Bootstrap design tokens:

// your_app/static/bootstrap_variables.scss
$primary:   #2c5f2e;
$secondary: #606c38;
$info:      #0077b6;

$font-size-base:         0.9rem;
$font-family-sans-serif: "Inter", system-ui, sans-serif;
 {# ------------------------------------------------------------------ #} {# ENTRYPOINT 2 #} {# ------------------------------------------------------------------ #}

Use this file for all AdminLTE-specific variables. Because Bootstrap variables are already resolved when this file is loaded, you can safely reference them: 

// your_app/static/adminlte_variables.scss
$lte-sidebar-color: $primary;      // references your $primary from step 2a
$lte-sidebar-bg:    $gray-900;
$lte-sidebar-width: 280px;
